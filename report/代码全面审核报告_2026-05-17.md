## 二、后端安全审核（Critical）

### 🔴 P0-1：数据范围上下文缺省返回"ALL"

**位置**：`DataScopeContext.java:36-39`

```java
public static String getDataScope() {
    DataScopeInfo info = CONTEXT.get();
    return info != null ? info.getDataScope() : "ALL";  // 默认是全部！
}
```

**问题**：当 ThreadLocal 为空时（请求未经过拦截器或拦截器异常），默认返回"ALL"，攻击者可通过删除 Header 绕过所有数据隔离。

**修复方案**：
```java
public static String getDataScope() {
    DataScopeInfo info = CONTEXT.get();
    if (info == null) {
        throw new SecurityException("数据权限上下文未初始化，请检查请求链路");
    }
    return info.getDataScope();
}
```

### 🟡 P1-1：SecurityConfig 只检查登录状态，未校验角色

**位置**：`SecurityConfig.java:35-38`

```java
.authorizeHttpRequests(auth -> auth
    .requestMatchers(publicPaths).permitAll()
    .anyRequest().authenticated()  // 只要登录就能访问所有接口

```

**问题**：任何登录用户都能访问所有接口，角色检查依赖 Controller 层 `@RequireRole` 注解，但该注解未在所有敏感接口上使用。

## 四、前端代码审核

### 🔴 F0-1：localStorage 敏感数据未加密

**位置**：`stores/user.ts`

```typescript
function setToken(value: string) {
  token.value = value
  localStorage.setItem(STORAGE_KEYS.token, value)  // 明文存储
}
```

**风险**：XSS 攻击可直接读取 token 和用户信息。

**修复方案**：使用 CryptoJS 加密存储：
```typescript
import CryptoJS from 'crypto-js'
function setToken(value: string) {
  token.value = value
  const encrypted = CryptoJS.AES.encrypt(value, SECRET_KEY)
  localStorage.setItem(STORAGE_KEYS.token, encrypted.toString())
}
```
