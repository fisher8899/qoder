角色与功能矩阵 — 月度业绩考核

概览 业务流程 角色功能 时序图

4 个核心角色 · 全流程协作

月度业绩考核业务围绕"指标制定 → 自评 → 互评 → 复核 → 申诉 → 发布"全流程展开，由 4 个角色按职责分工协同完成。系统管理员负责基础配置，业绩考核管理员（条件财务处）是全流程总指挥，部门绩效管理员负责本部门自评和互评录入，部门绩效考核员/分管领导对其他部门进行打分评估。

系
系统管理员
System Administrator
基础配置 · 全局权限
负责系统基础配置：用户、角色、权限、菜单、组织架构、考核单位、单据编码、系统参数等。不参与具体业务流程，但为所有业务运行提供基础保障。

考
业绩考核管理员
Assessment Manager · 条件财务处
总指挥 · 全流程管控
条件财务处主导，月度考核业务的全流程管理者：发布指标、跟进自评、组织互评、复核评分、处理申诉、预发布与正式发布考核结果。每月按时间节点推进考核工作。

部
部门绩效管理员
Department Performance Admin
本部门自评 · 数据录入
各业务部门指定的绩效管理员，负责本部门月度考核指标的自评数据录入、自评说明编写、佐证材料上传，对其他部门进行互评打分，查看本部门历史考核成绩。

结果查看

评
部门绩效考核员 / 分管领导
Department Evaluator / Vice President
他评打分 · 评估反馈
由公司分管领导或委派的考核员担任，对所分管或交叉评估的其他部门进行月度打分，根据被考核部门的自评数据、工作完成情况进行客观评分并提供意见反馈。
