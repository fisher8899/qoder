BUG-006 故障链：@Select 注解 → MyBatis 生成 SELECT 语义 → JDBC executeQuery() → MySQL Connector/J 8.x 拒绝 DML → SQLException: "Can not issue data manipulation statements with executeQuery()" → 事务回滚 → GlobalExceptionHandler 返回 500 → 前端弹出"系统繁忙，请稍后重试"。新增（POST）不走 delete 分支所以不触发。
修复：BizIndicatorOrgMapper 与 BizIndicatorLeaderMapper 的 deleteByIndicatorId 注解由 @Select 改为 @Update。修复后整条链路 ①→④ 正常走通。

审批流程（三级审批）
状态机：DRAFT → PENDING_DEPT_LEADER → PENDING_SUPERVISOR → PENDING_FINANCE → APPROVED
被退回：任意节点 → REJECTED（带 rejectReason），退回后可重新编辑再提交。

审批角色与状态机

  • 部门负责人（DEPT_LEADER）：PENDING_DEPT_LEADER → PENDING_SUPERVISOR
  • 分管领导（SUPERVISOR）：PENDING_SUPERVISOR → PENDING_FINANCE
  • 财务管理员（FIN_ADMIN）：PENDING_FINANCE → APPROVED
  • 退回操作（reject）：任意节点 → REJECTED，记录 rejectReason
